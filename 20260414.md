# **복습**

singleton : 클래스의 객체를 하나만 만들어서 계속 사용하는 것

flyweight : 클래스를 변하는 클래스(extrinsic) 와 변하지 않는 클래스(intrinsic)으로 구분하여 intrinsic 클래스는 singleton처럼 재사용한다.

prototype : 객체를 생성할 때 비용이 많이 드는 클래스를 clone하여 사용하는 것

## *bridge*

객체에서 동작을 처리하는 구현부와 확장을 위한 추상부를 분리한다.

**구조**

- Abstraction : **기능** 계층의 최상위 클래스이며 추상 인터페이스
- RefinedAbstraction : **기능** 계층에서 새로운 부분을 확장한 클래스
- Implementor : Abstraction의 기능을 **구현**하기 위한 인터페이스 정의
- ConcreteImplementor : 실제 기능을 **구현**하는 클래스

예시 

도형과 색상을 분리

구현부

```java
interface Color {
    void applyColor();
}

class Red implements Color {
    public void applyColor() {
        System.out.println("빨간색");
    }
}

class Blue implements Color {
    public void applyColor() {
        System.out.println("파란색");
    }
}
```

추상부(기능부)

```java
abstract class Shape {
    protected Color color; // 핵심: 포함 관계 (has-a)

    public Shape(Color color) {
        this.color = color;
    }

    abstract void draw();
}

class Circle extends Shape {
    public Circle(Color color) {
        super(color);
    }

    public void draw() {
        System.out.print("원 그리기 + ");
        color.applyColor();
    }
}

class Rectangle extends Shape {
    public Rectangle(Color color) {
        super(color);
    }

    public void draw() {
        System.out.print("사각형 그리기 + ");
        color.applyColor();
    }
}
```

main

```java
public class Main {
    public static void main(String[] args) {
        Shape redCircle = new Circle(new Red());
        Shape blueRectangle = new Rectangle(new Blue());

        redCircle.draw();     // 원 + 빨강
        blueRectangle.draw(); // 사각형 + 파랑
    }
}
```

조합이 많아질 때 구조를 분리해서 관리하기 위해 사용한다.
