# Templete Method!

> 디자인 패턴에서 템플릿은 변하지 않는 것을 의미한다.
> 

 여러 클래스에서 공통으로 사용하는 메서드를 **템플릿화** 해서 상위 클래스에 정의하고 하위 클래스에서 세부 동작을 다르게 구현하는 패턴.

장점 

- 로직을 공통화하여 코드의 중복을 줄일 수 있다.
- 서브 클래스의 역할을 줄이고, 핵심 로직을 상위 클래스에서 관리하므로 관리 용이
- 알고리즘 구조를 강제할 수 있음

단점

- 유연성의 제한
- 클래스의 생성, 관리의 어려움

예제

```java
// 추상 클래스 (Template)
abstract class Beverage {

    // Template Method (전체 알고리즘 고정)
    public final void make() {
        boilWater();
        brew();        // 자식이 구현
        pourInCup();
        addCondiments(); // 자식이 구현
    }

    private void boilWater() {
        System.out.println("물 끓이기");
    }

    private void pourInCup() {
        System.out.println("컵에 따르기");
    }

    // 변하는 부분
    protected abstract void brew();
    protected abstract void addCondiments();
}
```

```java
// 구현 클래스 (Coffee)
class Coffee extends Beverage {

    @Override
    protected void brew() {
        System.out.println("커피 추출");
    }

    @Override
    protected void addCondiments() {
        System.out.println("설탕 추가");
    }
}
```

```java
// 실행
public class Main {
    public static void main(String[] args) {
        Beverage coffee = new Coffee();
        coffee.make();
    }
}
```
