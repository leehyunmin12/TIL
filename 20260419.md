# index 란?

인덱스란 추가적인 쓰기 작업과 저장 공간을 활용하여 데이터베이스 테이블의 검색 속도를 향상시키기 위한 자료구조이다.

→ 책의 목차같은 느낌

인덱싱을 사용하는 이유?

테이블을 생성하고 데이터가 쌓이면, 테이블의 레코드는 내부적으로 순서가 없이 뒤죽박죽 저장된다.

이 상황에서는 레코드를 처음부터 끝까지 다 읽으면서 조건과 비교해야하는 Full Table Scan를 수행한다.

→ 데이터가 많을수록 비용이 많이 들고 성능 저하가 생김

인덱스 동작 방식?

테이블 생성 시, 생성하고 싶은 인덱스 컬럼을 지정한다.

특정 컬럼에 대한 인덱스를 생성하면, 해당 컬럼의 데이터들을 정렬하여 별도의 메모리 공간에 데이터의 물리적 주소와 함께 저장된다.

index의 종류

- B+tree Index
- Bitmap Index
- IOT Index
- Clutered Index

B+Tree 인덱스

밸런스 트리(Balanced Tree)란, 노드 삽입 및 삭제 시 특정 규칙에 맞게 재정렬 되어 양쪽 자식 노드 수의 밸런스를 유지하는 트리이다.

→ 밸런스를 유지하므로 항상 O(logN)의 시간복잡도를 가지기 때문에 매우 효율적임.

![image.png](image.png)

B+Tree는 리프 노드에만 데이터를 저장하고, 연결 리스트로 구성되어 있기에 범위 검색에 더욱 효율적이다.

리프 노드가 아니라면 자식의 key값을 저장합니다. 중복 키 발생 가능

하지만 삽입 삭제가 빈번히 일어나는 DB 경우 매번 재정렬 해야하는 index가 독이 될 수 있으니 주의하여 사용.
