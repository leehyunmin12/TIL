## **flyweight**

: 모든 객체를 인스턴스화 하지 않고 변하지 않는 객체는 재사용하여 최적화를 노리는 패턴이다. 경량패턴이라고도 함.

intrinsic : “고유한, 본질적인” 이라는 뜻으로 변하지 않는 인스턴스 → 공유 가능

extrinsic : “외적인, 비본질적인” 이라는 뜻으로 변할 수 있는 인스턴스 → 공유 불가능

사용 시기 

- 생성되는 객체의 수가 많아 저장 비용이 높을 때
- 생성된 객체가 오랫동안 메모리에 상주하여 사용되는 횟수가 많을 때
- 공통적인 인스턴스를 사용하는 로직이 포함된 경우
- 임베디드처럼 메모리를 최소한으로 사용해야 하는 경우

장점 

- 애플리케이션에 사용하는 메모리를 줄일 수 있다.
- 프로그램 속도 개선

단점

- 코드 복잡도 증가

예시 코드 (마인크래프트 나무)

```java
// ConcreteFlyweight - 플라이웨이트 객체는 불변성을 가져야한다. 변경되면 모든 것에 영향을 주기 때문이다.
final class TreeModel {
    // 메시, 텍스쳐 총 사이즈
    long objSize = 90; // 90MB

    String type; // 나무 종류
    Object mesh; // 메쉬
    Object texture; // 나무 껍질 + 잎사귀 텍스쳐

    public TreeModel(String type, Object mesh, Object texture) {
        this.type = type;
        this.mesh = mesh;
        this.texture = texture;

        // 나무 객체를 생성하여 메모리에 적재했으니 메모리 사용 크기 증가
        Memory.size += this.objSize;
    }
}
```

```java
// UnsahredConcreteFlyweight
class Tree {
    // 죄표값과 나무 모델 참조 객체 크기를 합친 사이즈
    long objSize = 10; // 10MB

    // 위치 변수
    double position_x;
    double position_y;

    // 나무 모델
    TreeModel model;

    public Tree(TreeModel model, double position_x, double position_y) {
        this.model = model;
        this.position_x = position_x;
        this.position_y = position_y;

        // 나무 객체를 생성하였으니 메모리 사용 크기 증가
        Memory.size +=  this.objSize;
    }
}
```

```java
// FlyweightFactory
class TreeModelFactory {
    // Flyweight Pool - TreeModel 객체들을 Map으로 등록하여 캐싱
    private static final Map<String, TreeModel> cache = new HashMap<>(); // static final 이라 Thread-Safe 함

    // static factory method
    public static TreeModel getInstance(String key) {
        // 만약 캐시 되어 있다면
        if(cache.containsKey(key)) {
            return cache.get(key); // 그대로 가져와 반환
        } else {
            // 캐시 되어있지 않으면 나무 모델 객체를 새로 생성하고 반환
            TreeModel model = new TreeModel(
                    key,
                    new Object(),
                    new Object()
            );
            System.out.println("-- 나무 모델 객체 새로 생성 완료 --");

            // 캐시에 적재
            cache.put(key, model);

            return model;
        }
    }
}
```

변할 수 있는 객체인 위치, 나무 모델 등의 필드를 가진 클래스와 (extrinsic)

변하지 않는 객체인 type, mesh, texture 등의 필드를 가진 클래스 (intrinsic)를 분리하고,

이들을 관리하는 클래스를 만들어서 intrinsic 객체는 한번만 만들고 재사용한다.

여기서 변화하지 않는 객체를 재사용하는 것은 singleton 패턴과 비슷하다.

주의할 점 : 위 TreeModelFactory에서는 HashMap을 활용하여 TreeModel 인스턴스를 캐싱하여 관리하고 있는데 ‘이처럼 관리되고 있는 인스턴스’는 GC(Garbage Collection)처리 되지 않는다는 것을 알고 있어야 한다!!!!!!!!!!!!
